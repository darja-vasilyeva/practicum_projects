## Описание проектов

Эти проекты были выполнены в ходе обучения в Яндекс.Практикуме (2024-2025).

| №  | Название                      | Описание                                                                                                                                                                                                                  | Технологии                                                     |
|----|-------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------|
| 01 | SWAPI                         | С помощью клиента для публичного API получить информацию о персонажах, планетах, звездолётах и других объектах вселенной «Звёздных войн» в формате JSON.                                                                    | Python, ООП, HTTP API, обработка исключений, сохранение данных на диск |
| 02 | GastroHub                     | Создать базу данных для управления сетью ресторанов, включая информацию о меню, заказах и клиентах; выполнить аналитические запросы.                                                                                        | SQL: оконные функции, вложенные подзапросы, фильтрация, агрегации |
| 03 | DWH для нескольких источников | Создать хранилище данных, способное интегрировать и обрабатывать информацию из различных источников. <br/> Задачи: <br/> 1) интеграция данных (PostgreSQL, MongoDB); <br/> 2) очистка и трансформация; <br/> 3) загрузка в DWH; <br/> 4) анализ и отчётность. | PostgreSQL, MongoDB, API, Python, SQL, Airflow, DAGs           |
| 04 | Handmade_marketplace          | Создать DWH из трёх источников данных (PostgreSQL) и витрин для маркетплейса товаров ручного производства. Подготовить две витрины: витрину отчётности по мастерам и витрину в разрезе одного месяца с учётом года. Использовать инкрементальную загрузку при обновлении витрины. Загрузить данные из нового источника (PostgreSQL) в хранилище, создать DDL для новой витрины. | СУБД PostgreSQL, DBeaver, инкрементальная загрузка             |
| 05 | ETL_for_homewares_retailer    | Разработать ETL-пайплайн для ритейлера, который продаёт в интернете товары для дома и автоматизировать ручные процессы из API и S3.                                                                                       | СУБД PostgreSQL, DBeaver, Docker, Airflow                      |
| 06 | Auto_Show_Bibip               | Создать базу данных для автосалона "Бибип", которую будут использовать приложения компании. <br/> Подзадачи: <br/> 1. Добавить автомобили и модели в базу данных <br/> 2. Сохранить продажи <br/> 3. Вывести список моделей, доступных к продаже <br/> 4. Вывести детальную информацию о машине <br/> 5. Обновить ключевое поле <br/> 6. Удалить продажу <br/> 7. Вывести топ-3 модели по количеству продаж в автосалоне | Python <br/> Основы языка: переменные, типы данных, условные операторы, циклы <br/> Работа с данными: списки, кортежи, множества, словари <br/> Функции: определение и вызов функций, параметры и аргументы <br/> Модули и библиотеки: импорт модулей, использование стандартных библиотек и сторонних пакетов <br/> Обработка ошибок: try-except блоки для обработки исключений <br/> Работа с файлами: чтение и запись данных в файлы.
